
**Multi-Agent Interview Coach с LangGraph**


LangGraph
Mistral AI
Python 3.12
Это многоагентная система на базе LangGraph для симуляции технического собеседования. Использует Mistral AI для генерации вопросов, анализа ответов, адаптации сложности и формирования итогового отчёта. Поддерживает раннее завершение, валидацию ввода и чистый JSON-лог.


**Данные**

AI.py - Python файл
AI.ipynb - ссылка на Google.Colab

**О проекте:**

Система имитирует реальное техническое интервью с несколькими агентами:

Interviewer — задаёт вопросы
Observer — анализирует ответы, ставит оценку (score 1–10), даёт скрытые рекомендации


**Особенности:**
Адаптивная сложность (вопросы усложняются/упрощаются в зависимости от score)
Скрытая рефлексия (мысли Observer → рекомендации Interviewer)
Автоматическое завершение после 10 вопросов или по команде «стоп интервью»
Проверка ввода данных кандидата (имя, позиция, грейд, опыт)
Полный JSON-лог сессии (вопросы, ответы, мысли агентов, отчёт)
Гарантированный отчёт даже при ошибках LLM

**Технологии:**

LangGraph — управление многоагентным workflow
LangChain + Mistral AI — генерация и анализ текста
Pydantic — структурированный вывод (structured output)
Python 3.12+

Необходимый API-ключ в коде:

os.environ["MISTRAL_API_KEY"] = "ваш_ключ_от_mistral.ai"

*Действия*

Запуск кода
Введите данные кандидата (система проверит формат):

textФИО кандидата: Иван Иванов
Позиция: Python Developer
Грейд: Middle
Опыт лет: 4

Отвечайте на вопросы или введите «стоп интервью» для завершения.
После завершения:
Отчёт выводится в консоль
Файл interview_log.json создаётся и скачивается (в Colab)


*Структура JSON-лога*
JSON{
  "participant_name_main": "Бухарев Святослав Андреевич",   // моё реальное ФИО (фиксировано)
  "participant_name": "Иван Иванов",                        // имя кандидата
  "turns": [

*Как работает система*

Ввод данных кандидата → валидация (повтор при ошибке)
Цикл интервью:
Interviewer → вопрос
Пользователь → ответ
Observer → анализ, score, скрытые мысли, обновление сложности

**Завершение:**

По команде «стоп интервью» или автоматически после 10 вопросов

Генерация отчёта (evaluator_prompt) → запись в лог
